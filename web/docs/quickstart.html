Quick Start Guide

Learn the basics of testing LLM agents with Senytl in just a few minutes.

Your First Test

Let's create a simple test for an LLM agent. First, install Senytl and then follow along:

Terminal 
pip install "senytl[pytest]"

Basic Agent Testing

Create a test file test_my_agent.py:

import pytest
from senytl import expect

@pytest.mark.senytl_agent
def test_simple_agent(senytl_agent):
    """Test a simple echo agent"""
    
    def my_agent(prompt: str) -> str:
        return f"Echo: {prompt}"
    
    # Wrap the agent for testing
    wrapped = senytl_agent(my_agent)
    
    # Run the agent
    response = wrapped.run("Hello, World!")
    
    # Make assertions
    expect(response).to_contain("Hello, World!")
    expect(response).to_start_with("Echo:")

Running Your Test

Run your test with pytest:

pytest test_my_agent.py -v

Run with coverage tracking:

pytest test_my_agent.py --senytl-coverage

Understanding the Response Object

The response object contains detailed information about the agent execution:

def test_response_object(senytl_agent):
    def my_agent(prompt: str) -> str:
        return f"Processed: {prompt}"
    
    wrapped = senytl_agent(my_agent)
    response = wrapped.run("test")
    
    # Access response properties
    print(f"Output: {response.output}")
    print(f"Success: {response.success}")
    print(f"Execution time: {response.execution_time}")
    print(f"Tokens used: {response.tokens_used}")
    
    # Make assertions
    expect(response.success).to_be_true()
    expect(response.output).to_contain("test")

Available Assertions

Senytl provides comprehensive assertions for testing agent responses:

Content Assertions

expect(response).to_contain("expected text")
expect(response).to_start_with("prefix")
expect(response).to_end_with("suffix")
expect(response).to_match_pattern(r"regex pattern")
expect(response).to_be_empty()
expect(response).to_have_length_greater_than(10)

Success and Status

expect(response.success).to_be_true()
expect(response.success).to_be_false()
expect(response.error).to_be_none()
expect(response.error).to_contain("specific error")

Performance Assertions

expect(response.execution_time).to_be_less_than(2.0)
expect(response.tokens_used).to_be_less_than(1000)
expect(response.cost_estimate).to_be_less_than(0.01)

Multiple Test Cases

Test your agent with various inputs using parameterized tests:

import pytest
from senytl import expect

@pytest.mark.senytl_agent
@pytest.mark.parametrize("input_text,expected_content", [
    ("hello", "hello"),
    ("test", "test"),
    ("python", "python"),
])
def test_agent_multiple_inputs(senytl_agent, input_text, expected_content):
    def my_agent(prompt: str) -> str:
        return f"You said: {prompt}"
    
    wrapped = senytl_agent(my_agent)
    response = wrapped.run(input_text)
    
    expect(response).to_contain(expected_content)
    expect(response.success).to_be_true()

Testing Error Cases

Test how your agent handles errors and edge cases:

@pytest.mark.senytl_agent
def test_agent_error_handling(senytl_agent):
    def problematic_agent(prompt: str) -> str:
        if prompt == "error":
            raise ValueError("Test error")
        return f"OK: {prompt}"
    
    wrapped = senytl_agent(problematic_agent)
    
    # Test normal case
    response = wrapped.run("hello")
    expect(response.success).to_be_true()
    
    # Test error case
    response = wrapped.run("error")
    expect(response.success).to_be_false()
    expect(response.error).to_contain("Test error")

Configuration

Configure Senytl behavior using a configuration file or environment variables:

pyproject.toml

[tool.senytl]
timeout = 30.0
max_retries = 3
default_model = "gpt-3.5-turbo"
log_level = "INFO"

[tool.senytl.providers.openai]
api_key = "your-api-key"
base_url = "https://api.openai.com/v1"

[tool.senytl.providers.anthropic]
api_key = "your-anthropic-key"

Environment Variables

export SENYTL_TIMEOUT=30.0
export SENYTL_DEFAULT_MODEL="gpt-3.5-turbo"
export OPENAI_API_KEY="your-api-key"

Best Practices

  • Use descriptive test names: Make it clear what each test is validating
  • Test both success and failure cases: Ensure your agent handles errors gracefully
  • Keep tests independent: Each test should be able to run in isolation
  • Use coverage tracking: Monitor which parts of your agent are being tested
  • Parameterize similar tests: Avoid code duplication with parameterized tests
