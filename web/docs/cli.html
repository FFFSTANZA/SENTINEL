CLI Commands

Complete reference for Senytl command-line interface commands and options.

Overview

The Senytl CLI provides commands for test generation, coverage analysis, CI integration, and project management. All commands are prefixed with senytl.

Terminal 
senytl --help

Version

Display the installed version of Senytl:

senytl version

Output:

Senytl 1.0.0
Python 3.10.5
Platform: linux-x86_64

Test Generation

Generate Tests from Agent Code

Automatically generate test cases from your agent implementation:

# Generate tests for a single agent file
senytl generate tests --agent path/to/my_agent.py

# Generate tests and save to specific directory
senytl generate tests --agent path/to/my_agent.py --output tests/generated/

# Generate tests with specific template
senytl generate tests --agent path/to/my_agent.py --template comprehensive

# Generate tests for all agents in a directory
senytl generate tests --agent path/to/agents/ --recursive

Interactive Test Generation

Start an interactive session to generate tests:

senytl generate --interactive

This will prompt you to:

  • Select agent files to analyze
  • Choose test scenarios to generate
  • Configure test parameters
  • Save generated tests

Test Generation Options

# Generate with specific coverage target
senytl generate tests --agent my_agent.py --coverage-target 90%

# Include performance tests
senytl generate tests --agent my_agent.py --include-performance

# Include adversarial tests
senytl generate tests --agent my_agent.py --include-adversarial

# Generate only unit tests
senytl generate tests --agent my_agent.py --type unit

# Generate integration tests
senytl generate tests --agent my_agent.py --type integration

Coverage Analysis

Suggest Missing Tests

Analyze your current tests and suggest improvements:

# Analyze coverage and suggest improvements
senytl suggest-tests

# Analyze specific test file
senytl suggest-tests tests/test_my_agent.py

# Analyze with detailed output
senytl suggest-tests --verbose

# Generate suggestions in JSON format
senytl suggest-tests --format json --output coverage_suggestions.json

Coverage Report

Generate comprehensive coverage reports:

# Generate HTML coverage report
senytl coverage --format html --output coverage_report.html

# Generate JSON coverage report
senytl coverage --format json --output coverage_data.json

# Generate markdown report
senytl coverage --format markdown --output coverage_report.md

# Show coverage summary in terminal
senytl coverage --summary

CI/CD Integration

Initialize CI Workflow

Set up continuous integration for your project:

# Initialize GitHub Actions workflow
senytl init-ci --github

# Initialize with custom configuration
senytl init-ci --github --config .senytl-ci.yml

# Initialize for GitLab CI
senytl init-ci --gitlab

# Initialize for other CI systems
senytl init-ci --template jenkins --output jenkinsfile

CI Configuration

Generate CI configuration files:

# Generate CI configuration
senytl ci-config --provider github --python-versions "3.9,3.10,3.11"

# Generate with coverage requirements
senytl ci-config --provider github --min-coverage 80 --max-latency 2.0

# Generate for multiple providers
senytl ci-config --provider github,gitlab,jenkins

Project Management

Initialize Project

Set up a new Senytl project:

# Initialize in current directory
senytl init

# Initialize in specific directory
senytl init my-llm-project

# Initialize with template
senytl init --template basic my-project
senytl init --template comprehensive my-project

# Initialize with configuration
senytl init --config custom_config.yml

Configuration Management

Manage project configuration:

# Show current configuration
senytl config show

# Set configuration value
senytl config set timeout 30.0
senytl config set providers.openai.api_key "your-key"

# Export configuration
senytl config export --output config.yml

# Import configuration
senytl config import config.yml

Testing Commands

Run Tests

Run your test suite with various options:

# Run all tests
senytl test

# Run specific test file
senytl test tests/test_my_agent.py

# Run tests with coverage
senytl test --coverage

# Run tests with performance tracking
senytl test --performance

# Run tests in parallel
senytl test --parallel 4

# Run tests with specific markers
senytl test --markers "senytl_agent and not slow"

# Run tests and generate report
senytl test --report html --output test_report.html

Performance Testing

Run performance benchmarks and load tests:

# Run performance tests
senytl performance

# Run load tests with specific parameters
senytl load-test --concurrent-users 10 --duration 300 --ramp-up 60

# Generate performance report
senytl performance --report --output performance_report.html

# Run SLA validation
senytl performance --sla-check --max-latency 2.0 --min-throughput 10

Utility Commands

Environment Management

# Check environment setup
senytl check-env

# Validate configuration
senytl validate-config

# Test API connections
senytl test-connections

# List installed providers
senytl list-providers

Session Management

# List recorded sessions
senytl sessions list

# Playback recorded session
senytl sessions playback session-id-123

# Export session data
senytl sessions export session-id-123 --output session_data.json

# Clean old sessions
senytl sessions clean --older-than 30d

Advanced Options

Logging and Debugging

# Enable verbose logging
senytl --verbose test

# Enable debug mode
senytl --debug test

# Save logs to file
senytl test --log-file test_logs.txt

# Set log level
senytl --log-level DEBUG test

Output Formatting

# Generate output in specific format
senytl coverage --format json --pretty

# Generate compact output
senytl test --output compact

# Generate machine-readable output
senytl test --output json --quiet

Configuration File

CLI options can be configured in pyproject.toml or .senytl.yml:

[tool.senytl.cli]
default_timeout = 30.0
parallel_workers = 4
log_level = "INFO"
output_format = "text"

[tool.senytl.cli.coverage]
min_coverage = 80.0
output_format = "html"
output_dir = "coverage_reports"

[tool.senytl.cli.test]
default_markers = ["senytl_agent"]
parallel_execution = true
failure_threshold = 0

Exit Codes

Senytl CLI commands return standard exit codes:

  • 0 - Success
  • 1 - General error
  • 2 - Invalid arguments
  • 3 - Configuration error
  • 4 - Test failure
  • 5 - Coverage requirement not met
  • 6 - Performance SLA violation
